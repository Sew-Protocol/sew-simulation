# Silent River: SEW Withdrawal Security Analysis Campaign

## Objective
Conduct a systematic adversarial analysis of withdrawal, bond-locking, slashing, and escrow-fund recovery behaviours in the SEW Protocol simulator.

## Key Findings from Code Review

### Withdrawal Actions Available
1. **`withdraw_stake`** - Resolver bond withdrawal (sew.clj:222-237)
   - Guards: active disputes block withdrawal, frozen resolvers blocked
   - **POTENTIAL GAP**: No check for pending slashes!

2. **`withdraw_escrow`** - Escrow principal withdrawal (sew.clj:239-247)
   - Guards: must be terminal state, must have claimable balance
   - Checks: `terminal-state?` and claimable balance > 0

3. **`withdraw_fees`** - Protocol fee withdrawal (sew.clj:249-256)
   - Guards: total-fees[token] > 0

### Key State Fields for Analysis
- `:resolver-stakes` - Current resolver stakes
- `:resolver-frozen-until` - Frozen resolvers (unix timestamp)
- `:pending-fraud-slashes` - Pending slash proposals
- `:resolver-slash-total` - Realized slash amounts
- `:claimable` - Escrow funds awaiting manual claim
- `:total-held` - Total protocol-held balances
- `:escrow-transfers` - All escrow states
- `:bond-balances` - Appeal bonds posted

**Target**: Withdrawal during slash window

- [ ] **CRITICAL**: Withdraw with pending slash (no guard exists!)

**CONFIRMED BUG**: `withdraw_stake` has NO check for pending slashes!

**Vulnerability Details**:
```clojure
;; sew.clj:222-237 - apply-action "withdraw_stake" 
(defmethod apply-action "withdraw_stake"
  [{:keys [agent-index]} world event]
  (let [ar (resolve-address agent-index (:agent event))]
    (if-not (:ok ar)
      ar
      (let [resolver-addr (:address ar)
            amount       (get-in event [:params :amount])]
        (cond
          (has-active-dispute-for-resolver? world resolver-addr)
          (t/fail :active-disputes-block-withdrawal)  ; ← Only checks :disputed state!
          
          (> (get-in world [:resolver-frozen-until resolver-addr] 0) (:block-time world))
          (t/fail :resolver-frozen)  ; ← Only checks frozen status!
          
          :else
          (reg/withdraw-stake world resolver-addr amount)))))  ; ← No pending-slash check!
```

**Attack Scenario**:
1. Resolver stake = 1000, assigned to escrow with amount 5000
2. Dispute raised, resolver makes biased decision
3. `propose_fraud_slash` creates pending slash: {:status :pending, :amount 500}
4. Resolver calls `withdraw_stake` with amount 1000
5. Current code allows withdrawal (no pending-slash check!)
6. Stake becomes 0
7. `execute_fraud_slash` tries to slash 500, but `slash-resolver-stake` caps at current stake:
   ```clojure
   ;; registry.clj:64-77
   (defn slash-resolver-stake [world resolver-addr amount]
     (let [current (get-stake world resolver-addr)
           actual  (min current amount)]  ; ← Only slashes what's available!
   ```
8. **Result**: Slash only recovers 0 instead of 500! Resolver escapes penalty!

**Fix Required**: Add pending-slash check to `withdraw_stake`:
```clojure
;; Add to sew.clj:222-237
(let [pending-slash-amount (reduce + 0
                         (for [[_ slash] (:pending-fraud-slashes world)
                               :when (and (= (:resolver slash) resolver-addr)
                                            (#{:pending :appealed} (:status slash)))]
                           (:amount slash)))]
  (when (> pending-slash-amount (- current amount))
    (t/fail :pending-slash-blocks-withdrawal)))
```

## Recommended New Invariants

1. **`pending-slash-locks-bond`**: Resolver with pending slash cannot withdraw stake
2. **`bond-coverage-sufficient`**: Total bond >= max possible slash for active disputes
3. **`withdrawal-idempotent`**: Successful withdrawal cannot be repeated
4. **`claimable-accounting-valid`**: Claimable balances match terminal escrow amounts
5. **`no-withdrawal-during-dispute`**: No escrow withdrawal while dispute active
6. **`accounting-conservation`**: Sum of all balances remains constant
